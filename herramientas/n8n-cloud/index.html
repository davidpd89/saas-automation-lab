Para quién es n8n

  • Consultores técnicos que necesitan control sobre APIs, logs y flujos complejos.
  • Solopreneurs con base técnica que procesan datos sensibles y quieren auto-hosting.
  • Equipos pequeños que quieren escalar sin que el coste de la herramienta crezca proporcionalmente.

n8n no es la mejor opción para coaches o formadores sin perfil técnico que quieren flujos rápidos sin configuración. Para ese caso, Make es más adecuado.

Cloud vs self-hosted: la diferencia real

n8n ofrece dos modalidades:

  • n8n Cloud: gestionado por n8n. Sin servidor propio. Starter desde 20 €/mes (2.500 ejecuciones); Pro desde 50 €/mes (10.000 ejecuciones).
  • n8n self-hosted: instalas n8n en tu servidor. El software es gratuito bajo licencia Sustainable Use, pero el servidor, la seguridad y el mantenimiento corren por tu cuenta.

Importante: self-hosted no es “gratis para todo”. La licencia limita el uso si redistribuyes n8n como producto o lo explotas como servicio comercial para clientes. Si tienes dudas sobre tu caso de uso, revisa la documentación de licencia antes de comprometerte.

Que hemos podido comprobar

  • Control total sobre APIs, lógica y datos: puedes hacer casi cualquier cosa.
  • Self-hosting disponible: procesa datos en tu propio servidor.
  • Cobra por ejecuciones completas, no por pasos: más predecible con flujos largos.
  • Open-source con comunidad activa y cientos de nodos personalizados.
  • Excelente para IA: integra modelos de lenguaje, RAG y agentes de forma nativa.

Contras y riesgos

  • Curva de aprendizaje alta: requiere entender JSON, expresiones y algo de lógica técnica.
  • Self-hosting no es gratis: servidor, mantenimiento, seguridad y actualizaciones corren por tu cuenta.
  • La licencia Sustainable Use tiene restricciones si usas n8n como producto/servicio para clientes.
  • Debugging más complejo que en Make para flujos largos.
